# Reglas de pruebas

- Nombrar tests con intención clara.
- Usar fixtures estables y reproducibles.
- Añadir pruebas para cada contrato o regresión detectada.
- Bloquear cierre de hito si falla cualquier regresión acumulada.
- Solo cuando una batería de cierre de hito esté completamente en verde puede actualizarse `README.md`.
